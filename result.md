本文件汇总 `experiments/` 下全部 `test_metrics_*.txt` 的测试结果，共 15 份。所有结果均记录为 100 张测试图像上的 Dice、IoU、Precision 和 Recall；数值越大越好。除“批大小对比”外，表中对比均保持 `batch_size=8`。本轮不重新训练模型，只整理已保存的测试输出。

实验组别：A = U-Net + BCE；B = U-Net + BCE + Dice；C = Attention U-Net + BCE；D = Attention U-Net + BCE + Dice。

## 7. 初步结论

1. 在相同 `batch_size=8`、`lr=1e-4` 下，D 组取得最高 Dice（0.8265）、IoU（0.7083）和 Precision（0.8505），相对 A 组 Dice 提升 0.0378。当前最适合作为以区域重叠和误检控制为主的最终方案。
2. B 组的 Recall 为 0.8446，高于 D 组的 0.8134，说明在当前阈值和训练条件下，加入 Attention 后并未带来最高的检出率。若任务更重视减少漏检，可优先考虑 B 组，或进一步调整 D 组阈值与损失权重。
3. 混合损失本身有效：B 组相对 A 组的 Dice、IoU、Precision 和 Recall 分别提高 0.0299、0.0403、0.0129 和 0.0461。注意力机制本身也提高了 Dice 和 Recall，但 C 组 Precision 略低于 A 组。
4. 对 B、C、D 三类配置，`lr=1e-4` 的 Dice 均为各自最佳；只有 A 组在 `lr=1e-3` 达到最佳 Dice。因此学习率应按模型和损失配置选择，不能仅按单一基线结果固定。
5. B 组中 `BCE:Dice=2:1` 的 Dice、IoU 与 Precision 最佳，`1:1` 的 Recall 最佳；将 Dice 权重提高至 `1:2` 反而使四项指标均下降。后续可围绕 `1:1` 到 `2:1` 做更细粒度搜索，并对 D 组重复权重扫描。
6. 当前结论来自一次固定随机种子划分的 100 张测试图像，尚未包含重复训练、交叉验证或显著性检验。报告中宜表述为“当前实验设置下的初步结果”，而非普适性能结论。
